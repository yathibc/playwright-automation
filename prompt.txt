RCB Ticket Automation v2.0 — API-First Hybrid Approach

OBJECTIVE: Automate RCB ticket booking on shop.royalchallengers.com using Playwright.
Speed is critical — tickets sell out in ~2 minutes. No unnecessary waits.

ARCHITECTURE: Hybrid — API calls for polling/data, browser UI for login/canvas/checkout/payment.
All timeouts are config-based (env vars or .env file).

FLOW (8 phases):

Phase 1: Login & Token Extraction
- Open https://shop.royalchallengers.com
- Login with phone + manual OTP (timeout: OTP_WAIT_MINUTES=5)
- Extract rtokn cookie for API auth

Phase 2: Event Discovery (API-only)
- Poll GET /ticket/eventlist/O with Bearer rtokn
- Find event where event_Name contains both target teams AND event_Button_Text = "BUY TICKETS"
- Retry every API_POLL_INTERVAL_MS=3000 until found or EVENT_POLL_MINUTES=60 timeout

Phase 3: Navigate to Event Page
- Go to /ticket/{event_Code} with waitUntil:domcontentloaded
- Apply browser zoom to 50% (document.body.style.zoom = '0.5')
- Start Konva canvas interceptor
- Dismiss Continue popup if present

Phase 4: Stand Selection
- For each stand in STAND_PRIORITY (BOAT C STAND,C STAND,BOAT B STAND,B STAND):
  - Click stand via XPath: //p[text()='CATEGORY']/following-sibling::div[1]//p
  - Select ticket count (2)
  - Click Continue
  - Wait for intercepted seat-template + seatlist data

Phase 5: Seat Selection (Konva Canvas)
- KonvaSeatMapResolver: merge template + availability, calculate coordinates
- Find N consecutive seats in same row
- Convert to browser coords (account for canvas rect + Konva scale + zoom)
- Click seats on canvas via page.mouse.click()
- Click Proceed

Phase 6: Handle ticketaddtocart Response
- Intercept via page.waitForResponse()
- Success → Phase 7
- SEAT NOT AVAILABLE → retry new seats (same stand)
- STAND LIMIT EXCEEDED → try next stand
- MATCH/TRANS/PROFILE/USER LIMIT → hard stop

Phase 7: Addon Selection
- If addon modal: click "Free Metro Ticket", skip parking, click Continue
- If no modal: auto-navigates to /checkout

Phase 8: Checkout & Payment (relaxed, 8+ min)
- Fill checkout details, accept terms, PAY NOW
- UPI or Card payment via Juspay iframe
- Wait for payment completion (PAYMENT_WAIT_MINUTES=10)

KEY FILES:
- src/index.js — Main 8-phase orchestration
- src/config/config.js — All configuration + timeouts
- src/browser/browser.js — Browser + zoom + auth token
- src/detection/matchDetector.js — API event polling
- src/utils/konvaSeatMapResolver.js — Konva coordinate calculation
- src/utils/konvaCanvasInterceptor.js — Network interception
- src/flows/checkoutFlow.js — Checkout + addon + payment
- src/auth/login.js — OTP login

SPEED RULES:
- Never use waitForTimeout() unless absolutely necessary
- Use page.waitForResponse() for API interception
- Use domcontentloaded not networkidle
- Browser zoom 50% to avoid canvas scrolling
- All timeouts from config, never hardcoded

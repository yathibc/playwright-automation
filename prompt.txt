SEAT CONFIRMATION AND RETRY LOGIC
==================================================

After selecting seats and clicking Next / Continue / Confirm, the system may show errors such as:

- seat already selected by someone else
- seat not available
- selection failed
- unable to proceed
- or navigation does not proceed to the next page

In such cases, implement a retry mechanism controlled by CONFIG.SEAT_RETRY.

Retry rules:
1. Start a retry timer immediately after the first failed confirmation attempt.
2. Continue retrying for up to CONFIG.SEAT_RETRY.MAX_RETRY_DURATION_MINUTES.
3. Retry only within the currently selected stand. Do not switch to a different stand during the retry phase unless explicitly unavoidable due to total stand exhaustion and only after logging that reason.
4. Each retry cycle should:
   - detect failure using visible error messages, toasts, modal messages, inline warnings, or absence of navigation to the next page
   - clear previous selection if required
   - re-fetch the latest seat map and availability state
   - avoid previously failed seats if identifiable
   - quickly re-select another valid consecutive pair in the same stand
   - click Next / Continue / Confirm again
5. During retry:
   - prioritize speed over perfection
   - if CONFIG.SEAT_RETRY.RELAX_ROW_PREFERENCE_DURING_RETRY is true, relax row preference as needed
   - prioritize any valid consecutive seats over ideal rows once retries begin
6. Between retries:
   - wait CONFIG.SEAT_RETRY.RETRY_DELAY_SECONDS
   - allow UI to stabilize
7. For each retry cycle:
   - log attempt number
   - log stand name
   - log row/seat identifiers if available
   - log visible failure reason if any
   - send meaningful Telegram retry updates without spamming
8. Stop retrying when:
   - navigation to the next page is successful
   - or retry duration exceeds CONFIG.SEAT_RETRY.MAX_RETRY_DURATION_MINUTES
9. Success criteria for retry:
   - cart/summary/checkout page loads
   - or there is clear evidence that seat selection has been accepted and the flow has advanced
10. If successful after retry:
   - send Telegram alert "Seat confirmed after retry"
   - include stand, row, and seat details if available
11. If all retries fail:
   - take screenshot
   - log "Seat selection failed after retries"
   - send Telegram alert
   - end the process normally
